## Critical implementation rules

- `export const runtime = 'nodejs'` MUST be the VERY FIRST LINE — before ALL imports (DEPL-04)
- CRON_SECRET comparison: `request.headers.get('authorization') !== \`Bearer \${process.env.CRON_SECRET}\`` — return 401 immediately if mismatch
- NULL filter: use `.is('sms_sent_at', null)` NOT `.eq('sms_sent_at', null)` — Supabase requires `.is()` for NULL comparisons
- Batching: group alerts by `${subject}:${course_number}` using a Map BEFORE any Banner calls
- Re-alert reset: separately query alerts where `is_active = false` AND `sms_sent_at IS NOT NULL` AND `sms_opted_out = false` — for each, check Banner; if seatsAvailable === 0, reset sms_sent_at to null and is_active to true
- Twilio error 21610: catch all errors from sendSeatAlert, check `(err as { code?: number }).code === 21610`; if true, set `sms_opted_out = true`, do NOT set sms_sent_at; for other errors, log and continue (don't crash cron)
- After SMS success: set `sms_sent_at = new Date().toISOString()`, `is_active = false`, `sms_sid = sid` in one `.update()` call
- Return `{ checked: number, alerted: number }` — checked = total alerts processed, alerted = SMS sent count

```typescript
// Imports lib/twilio.ts which requires Node.js native modules.
// Edge Runtime is not supported — declare nodejs runtime explicitly.
export const runtime = 'nodejs'

import { NextRequest, NextResponse } from 'next/server'
import { adminClient } from '@/lib/supabase/admin'
import { getSeatsByCRN } from '@/lib/banner'
import { sendSeatAlert } from '@/lib/twilio'

export async function GET(request: NextRequest): Promise<NextResponse> {
  // 1. Authenticate via CRON_SECRET (ALRT-04)
  const auth = request.headers.get('authorization')
  if (auth !== `Bearer ${process.env.CRON_SECRET}`) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
  }

  // 2. Fetch all active alerts not yet notified and not opted out (ALRT-05)
  const { data: alerts, error: fetchError } = await adminClient
    .from('alerts')
    .select('id, crn, subject, course_number, course_name, phone_number, sms_opted_out')
    .eq('is_active', true)
    .is('sms_sent_at', null)
    .eq('sms_opted_out', false)

  if (fetchError) {
    console.error('[cron/check-seats] Failed to fetch alerts:', fetchError.message)
    return NextResponse.json({ error: 'Database error' }, { status: 500 })
  }

  if (!alerts || alerts.length === 0) {
    return NextResponse.json({ checked: 0, alerted: 0 })
  }

  // 3. Group alerts by (subject, course_number) to batch Banner calls (ALRT-06)
  const groups = new Map<string, typeof alerts>()
  for (const alert of alerts) {
    const key = `${alert.subject}:${alert.course_number}`
    if (!groups.has(key)) groups.set(key, [])
    groups.get(key)!.push(alert)
  }

  let checked = 0
  let alerted = 0

  // 4. For each group, deduplicate CRNs and call Banner once per unique CRN (ALRT-06)
  for (const [, groupAlerts] of groups) {
    const { subject, course_number } = groupAlerts[0]

    // Deduplicate CRNs within the group — multiple subscribers may watch the same section.
    // This is the batching core: 50 subscribers on 3 CRNs = 3 Banner calls, not 50.
    const uniqueCrns = [...new Set(groupAlerts.map((a) => a.crn))]

    for (const crn of uniqueCrns) {
      // ONE Banner API call per unique CRN regardless of subscriber count (ALRT-06 batching)
      let seatData
      try {
        seatData = await getSeatsByCRN(crn, subject, course_number)
      } catch (err) {
        console.error(`[cron/check-seats] Banner error for CRN ${crn}:`, err)
        continue
      }

      // All subscribers watching this specific CRN
      const crnAlerts = groupAlerts.filter((a) => a.crn === crn)
      checked += crnAlerts.length

      if (!seatData || seatData.seatsAvailable <= 0) continue

      // Seat is open — dispatch SMS to ALL subscribers of this CRN (ALRT-07)
      for (const alert of crnAlerts) {
        try {
          const sid = await sendSeatAlert(
            alert.phone_number,
            alert.course_name ?? `${subject} ${course_number}`,
            alert.crn
          )
          // Update alert: mark sent, deactivate, store SID (ALRT-09)
          await adminClient
            .from('alerts')
            .update({
              sms_sent_at: new Date().toISOString(),
              is_active: false,
              sms_sid: sid,
            })
            .eq('id', alert.id)
          alerted++
        } catch (err: unknown) {
          // Twilio error 21610 = recipient opted out (STOP reply) — permanent block (ALRT-10)
          if ((err as { code?: number }).code === 21610) {
            await adminClient
              .from('alerts')
              .update({ sms_opted_out: true })
              .eq('id', alert.id)
          } else {
            console.error(`[cron/check-seats] Twilio error for alert ${alert.id}:`, err)
          }
        }
      }
    }
  }

  // 5. Re-alert reset: for previously notified alerts, reset if seat has closed
  // This allows re-alerting if the seat opens again after being filled
  const { data: notifiedAlerts } = await adminClient
    .from('alerts')
    .select('id, crn, subject, course_number')
    .eq('is_active', false)
    .not('sms_sent_at', 'is', null)
    .eq('sms_opted_out', false)

  if (notifiedAlerts) {
    for (const alert of notifiedAlerts) {
      try {
        const seatData = await getSeatsByCRN(alert.crn, alert.subject, alert.course_number)
        if (seatData && seatData.seatsAvailable === 0) {
          // Seat closed again — reset so subscriber is re-alerted when it reopens
          await adminClient
            .from('alerts')
            .update({ sms_sent_at: null, is_active: true })
            .eq('id', alert.id)
        }
      } catch {
        // Non-fatal — skip this alert on error, try again next cron run
      }
    }
  }

  return NextResponse.json({ checked, alerted })
}
```

Implementation notes:
- `export const runtime = 'nodejs'` MUST be line 1 — before all imports (DEPL-04)
- CRON_SECRET check uses exact string comparison — returns 401 immediately if wrong
- `.is('sms_sent_at', null)` — Supabase requires `.is()` for NULL comparison, NOT `.eq()`
- `.not('sms_sent_at', 'is', null)` — Supabase syntax for IS NOT NULL

- `course_name ?? \`${subject} ${course_number}\`` — graceful fallback for SMS if course_name not stored yet
- Each Banner/Twilio call is wrapped in its own try/catch so one failure does not abort the whole cron run

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | Authorization: Bearer header validation | mitigate | Route checks `auth !== \`Bearer ${CRON_SECRET}\`` and returns 401 immediately; no processing occurs without correct secret |
| T-03-03 | Tampering | Double-send prevention — sms_sent_at gate | mitigate | Cron query filters `sms_sent_at IS NULL` — once set, alert is excluded from all future cron runs until seat closes |
| T-03-04 | Disclosure | SUPABASE_SERVICE_ROLE_KEY in adminClient | mitigate | adminClient is server-only; import 'server-only' in admin.ts prevents client bundle inclusion; key never NEXT_PUBLIC_ prefixed |
| T-03-12 | Denial of Service | Slow Banner API causing cron timeout | accept | Each getSeatsByCRN call has 10s timeout (set in Phase 2); Vercel function default timeout is 10s for Hobby / 60s for Pro; batching by course reduces total calls; individual failures skip (continue) without aborting |
| T-03-13 | Elevation | CRON_SECRET not set in production | mitigate | If CRON_SECRET env var is undefined, `Bearer undefined` would match any request with `Authorization: Bearer undefined`; operator must ensure CRON_SECRET is set in Vercel project env vars before deployment |
</threat_model>
